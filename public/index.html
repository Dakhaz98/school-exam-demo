Server URL

Use this only when this browser tab is not opened from the same address as the running app (for example a preview on another port). Paste the platform URL, click Test & save, and the page will refresh once.

Welcome

Administration uses admin as the user id. Teachers use their Staff ID from the roster. Students use their Student ID. The exam lobby opens for a window before the scheduled start until the end time.

If lists or uploads do not load, open Server URL in the header, enter the address where the platform runs, and use Test & save. No command line is required.

Quick trial

On the Rosters & Excel tab, click Load AIS trial scenario to preload the platform, then use Fill: Student-1 / 2 / 3 and Fill: teacher-1 / 2 below (one device per student id).

Three-tab exam rehearsal (after Load AIS trial scenario)
  1. Open three tabs to the same site address (for example three times Ctrl+T). Each tab remembers its own role.
  2. Tab 1 – Administration: Load a demo scenario on Rosters & Excel, then use Live control → Open command center on a room: a new tab opens with the full room dashboard (same tools as the teacher view). If the pop-up is blocked, the command center opens in the current tab instead.
  3. Tab 2 – Teacher: Quick fill teacher-1 (or teacher-2) → Continue → Proctor desk → Join my assigned room when the top banner allows access. Use Student admission & paper release: admit each student when you confirm presence, then Release question paper (required after admit — without it, questions stay hidden). You will see camera feeds, the room log, and integrity flags.
  4. Tab 3 – Student: Quick fill Student-1 (or 2 / 3 on other devices) → Continue → Student desk → allow camera and microphone → Begin exam. You will see a waiting area first; after admit and paper release, the exam page shows questions one at a time (Submit answer / Finish exam).

If your browser or Windows blocks the download

SmartScreen and some antivirus tools may warn about Office files from a local server. Try the CSV template link below (same columns, often fewer warnings). After download, you can open the CSV in Excel and use Save As to create an .xlsx if you prefer. On Windows: right-click the file, Properties, then press Unblock if you see it, or use Keep / Show more in the browser download bar.

Professional capability map (this build vs roadmap)

Source of truth: /api/platform/status (JSON for procurement).

In this build

    Roadmap (production)

      Question papers (upload before choosing a model)

      After the student roster is on the platform, you can upload your own paper here. Download the template, fill it in Excel, then upload. The new paper appears in the Question model list below with an [Uploaded] tag. Demo papers stay available as [Demo].

      Download question Excel template

      File format: .xlsx or .xls (first sheet only). One row per question. Columns (header row, names are matched flexibly): Question ID (optional), Question Text (required), then Choice A … Choice D, and optional Correct (letter A–D, number 1–4, or exact option text) for automatic multiple-choice scoring. You need at least two non-empty choices; three or four are fine. This demo stores multiple-choice answers only (no essays or images).

      Uploaded papers

      Create exam session

      Order: upload rosters (Admin tab), optionally upload a question file above, then pick the grade (values come only from the Grade column in your student file), set rooms, and apply layout. Set schedule and pick a question model. You can assign proctors manually (teacher names appear) or use random fill. Publishing fills any missing slots from eligible teachers when possible, then checks each room.

      Question pool: set a number lower than the model size to give each student a different random subset (recommended pool is larger than the draw count). Same shuffle rules and Correct keys apply to issued items only.

      Rooms and proctors

      Set required proctors per room, then assign. Random uses only teachers whose supervised grade matches the exam grade.

      Publish

      Before publish, the server fills missing proctor slots from teachers whose supervised grade matches the exam, when enough staff exist. If you still see an error, lower "Proctors required" per room or add teachers for that grade in the roster.

      Session timing

      Exam access key

      When a key is set, student browsers must enter the same value on the login screen before Continue.

      Safe Exam Browser (SEB)

      The server checks X-SafeExamBrowser-RequestHash against SHA256(fullRequestUrl + key) for each line (same rule as Moodle’s SEB quiz access). The URL must match what students type in the address bar (scheme, host, port, path). Behind a proxy, configure X-Forwarded-Host and X-Forwarded-Proto.
