# الخدمات المنطقية (Logical services)

المستودع يحتوي على **مسارين تنفيذيين** يتجهان لدمج SaaS؛ المسارات الفعلية أدناه حتى لا نضطر لنقل ملفات تكسر النشر قبل أوانه.

| الخدمة | الوصف | المسار الفعلي في المستودع | النشر الحالي (Render) |
|--------|--------|---------------------------|------------------------|
| **exam-realtime** | غرف الامتحان، المراقب، Socket.IO، رفع Excel، SQLite | جذر المستودع: `server.js`, `public/`, `lib/` | نعم — `render.yaml` → `npm start` |
| **console-web** | لوحة المؤسسة، الطلاب، الغرف، بناء النماذج، Supabase، LiveKit APIs | `web/` | لا (تشغيل محلي `npm run web:dev`؛ خدمة ويب ثانية لاحقاً) |

## اتجاه الدمج

1. توثيق عقد HTTP/WebSocket بين `console-web` و`exam-realtime` (انظر `docs/saas-scope-4d.md`).
2. إضافة خدمة Render ثانية لـ `web` عند توفر النطاق والمتغيرات.
3. اختياري: استخراج حزم مشتركة إلى `packages/` عند ظهور أنواع/تحقق مشترك بين الجذر و`web/`.

لا يوجد مجلد `services/exam-realtime/src` منفصل بعد — **تجنباً لكسر المسارات** في `package.json` وBlueprint؛ أي نقل فيزيائي يُقرَّر بعد CI أو سكربت بناء يحدّث `startCommand`.
