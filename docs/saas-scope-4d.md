# نطاق ٤ أيام — منتج SaaS (التزام تنفيذي)

هذا المستند يثبت: **ما يُشحن** في نافذة زمنية قصيرة، **ما لا يُكسر** كأساس، و**ما يُؤجَّل** بقرار صريح — بديلاً عن “حل تجريبي” يخفي الدين التقني.

---

## Stack (مؤكَّد من الكود الحالي + اتجاه التوسّع)

| طبقة | اختيار | ملاحظة |
|------|--------|--------|
| غرفة الامتحان + المراقب اللحظي | Node.js 22، Express، Socket.IO، SQLite (`lib/sqlite-store.js`) | الخدمة المنشورة حالياً على Render (`render.yaml` → `npm start`) |
| لوحة المؤسسة / الطلاب / الغرف / الفيديو | Next.js 15 (App Router)، React 19، Tailwind، TypeScript (`web/`) | جاهز للتطوير المحلي؛ النشر كخدمة ثانية عند الحاجة |
| هوية ومستأجرون (قاعدة SaaS) | Supabase (Auth + Postgres في `web/` — `schema.sql`، عميل SSR) | الأساس العالمي للمتعدد المستأجرين يبدأ هنا |
| بث الغرفة | LiveKit (رموز من `web/src/app/api/livekit/token`) | يتطلب مفاتيح بيئة؛ لا يُفترض تشغيله في الـ ٤ أيام إلا إن وُجدت المفاتيح |
| استضافة الإنتاج الحالي | Render (Blueprint) | خدمة ويب واحدة للـ realtime؛ إضافة خدمة `web` لاحقاً عند ربط النطاق والفوترة |

**قرار صريح:** لا نستبدل SQLite في غرفة الامتحان خلال هذه النافذة ما لم يُربط عقد API بين `web` و`exam-realtime`؛ الانتقال إلى Postgres للجلسات يبقى **مؤجلاً** إلى مرحلة الدمج.

---

## الأيام الأربعة (جدول يومي)

### اليوم ١ — تثبيت المنتج والمستودع

- **يشحن:** وثائق النطاق (هذا الملف)، خريطة `services/README.md`، توحيد تسمية الخدمات في README الجذر إن لزم.
- **أساس لا يُكسر:** مسار النشر الحالي (`npm ci` + `npm start`) يبقى أخضر؛ أي refactor لا يكسر `/api/health`.
- **مؤجل:** فصل `server.js` إلى حزمة منفصلة فعلياً (نقل ملفات) — يُخطط له بعد وجود اختبارات دخان أو CI.

### اليوم ٢ — حدود الخدمات وعقد API أولي

- **يشحن:** وثيقة عقد REST/WebSocket بين “لوحة Next” و“غرفة Express” — **`docs/api-contract-exam-realtime.md`** (مسودة `0.1`؛ تُوسَّع مع كل تغيير حرج).
- **أساس:** عدم خلط مسؤوليات الهوية: الطلاب في الغرفة يبقون على نموذج الجلسة الحالي حتى يُربط SSO/رموز قصيرة من Supabase.
- **مؤجل:** تنفيذ الربط الكامل (JWT متبادل، تدوير المفاتيح) إن لم يُنجز في اليوم ٣.

### اليوم ٣ — تكامل اختياري حسب البيئة

- **يشحن (إن وُجدت المتغيرات):** تدفق تسجيل دخول من `web` يفتح أو يوجّه نحو غرفة امتحان (رابط عميق، رمز جلسة).
- **يشحن (بدون LiveKit):** تحسين مسارات لوحة التحكم والامتحانات دون افتراض بث.
- **مؤجل:** MAP/NWEA الرسمي داخل المنصة — غير قانوني/غير واقعي بدون عقد NWEA؛ البديل المعتمد لاحقاً: **مرافق مراقبة** + روابط رسمية.

### اليوم ٤ — جودة الإنتاج والمراقبة الأولى

- **يشحن:** `npm run doctor` موسَّع أو سكربت فحص بيئة Render؛ تسجيل (`lib/logger.js`) متسق للأحداث الحرجة؛ قائمة تحقق نشر في `DEPLOY_RENDER.md` إن لزم.
- **أساس:** عدم إزالة دعم SEB/القيود الحالية دون بديل موثّق.
- **مؤجل:** فوترة Stripe، OIDC مؤسسي كامل، QTI، مراقبة APM مُدارة (Datadog إلخ).

---

## أساس لا يُكسر (ملخّص)

1. صحة النشر والـ health check.
2. فصل واضح بين **بيانات الغرفة اللحظية** و**بيانات المؤسسة طويلة الأمد** (حتى قبل الدمج الكامل).
3. عدم وعد تكاملات تقييم رسمية بدون عقود واضحة.

---

## مؤجَل صريح (لا يُعرض كـ “جاهز”)

- Postgres لجلسات الامتحان بدل SQLite قبل عقد API واختبار الحمل.
- خدمة Render ثانية لـ `web` + نطاق موحّد + CI متعدد الخدمات.
- MAP Growth كمنتج داخل المنصة.
- الفوترة والامتثال الكامل (GDPR/FERPA workflows) كمنتج وليس كملاحظة جانبية.
